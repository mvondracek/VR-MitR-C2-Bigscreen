Overview of attacks
  • Botnet-like group of infected Bigscreen applications that can be controlled by this command and control panel.
  • Force download and execution any payload (virus, malware, etc.) to victim's computer. RCE
  • Run program on victim's machine.
  • Remote REPL (remote Javascript eval) on victim's machines.JS RCE
  • Invisibly join any known Bigscreen room. Attacker is not visible in VR. Attacker's username is hidden from Room Preview UI and from Room Participants UI.
  • Receive victim's audio and video without victim's knowledge. Eavesdropping
  • Persistently eavesdrop victim's chat, even if they go to another room.
  • Ask user to download required fake driver. Phishing, RCE
  • Remotely kill victim's Bigscreen application. Denial-of-service
  • Force victim to send any chat message.
  • Ban selected user until restart. Denial-of-service
  • Kick user from room. Only admin should be able to do this. Denial-of-service
  • Set selected user as admin. Privilege escalation
  • Change room's settings VR locks. Only admin should be able to do this.
  • Redirect Bigscreen's UI to any webpage. Phishing
  • Gather all victim's logs.
  • Force victim to open screenshot directory.
  • Change user's avatar.
  • Play various sound effects.
Limitations of current implementation
  • Peer connection intended for multimedia transport currently does not support fallback to TURN servers. Support for STUN servers was sufficient for this proof of concept. With support for STUN servers, we were able to connect with users across Internet without complications.
  • Peer connection establishment is currently supported only with participants who are already in the room. This command and control panel does not initiate the connection but accepts incoming connections. This was sufficient for this proof of concept
  • Command and control panel currently does not support reconnecting to zombies after page refresh. Please restart relay server if you refresh this page.
